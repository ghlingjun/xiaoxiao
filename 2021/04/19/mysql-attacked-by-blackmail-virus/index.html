解决过程

1. 第一时间修改密码

  1. 修改服务器密码
  2. 修改数据库密码
  3. 修改数据库访问端口

2. 确认数据恢复条件

数据恢复需要具备两个条件：

  1. 有历史数据库备份文件；
  2. MySql服务开启了 binlog；
    若条件具备，才可进行数据恢复。

3. 恢复方法

1. 查看binlog信息

show variables like ‘%log_bin%’;

备份 binlog 文件，防止日志文件丢失，binlog 文件目录可能为：/usr/local/mysql/var，具体位置查询方式有两种：

  1. 查询 MySql 配置文件 my.cnf，查看 binlog 路径配置项
  2. 通过 sql 命令查询

2. 根据推断确认恢复内容

导出日志内容：

stop-datetime 为数据库被攻击时间；
